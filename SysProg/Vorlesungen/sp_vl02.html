Variablen, Operatoren, Kontrollfluss

Im Wesentlichen wie von Java gewohnt … :-)

  • Wichtig(st)e Abweichungen:
    1. ANSI-C: Deklaration aller Variablen am Anfang eines Blocks
    2. Im booleschen Kontext wird int als Wahrheitswert interpretiert: Alle Werte ungleich 0 entsprechen true (!)

=> Vorsicht mit

if (c=4) { ... }

Beispiel für Schleifen und Kontrollstrukturen in C:

} else {
    y = 2*x;

while (y>0) {

for (x=0; x<10; x++) {

Basisdatentypen

char Zeichen (ASCII, 8 Bit bzw. 1 Byte)
int Ganze Zahl (16, 32 oder 64 Bit)
float Gleitkommazahl (typ. 32 Bit)
double Doppelt genaue Gleitkommazahl (typ. 64 Bit)
void Ohne/kein Wert

Außerdem Arrays und Pointer mit diesen Typen möglich.

Und was ist mit booleschen Werten?

  • Es gibt kein bool (erst ab C99: als Makro)!

    => Nehmen Sie int …

    Alle Werte ungleich 0 werden im booleschen Kontext als true interpretiert.

    Achtung: Ein Rückgabewert von 0 wird in der Regel als Erfolg betrachtet!

Typmodifikatoren ändern Bedeutung

Vorangestellte Modifikatoren ändern Bedeutung:

  • Länge im Speicher

    short Speicher: halbe Wortlänge
    long Speicher: doppelte/dreifache Wortlänge

  • Vorzeichen

    signed mit Vorzeichen (Default bei Zahlen)
    unsigned ohne Vorzeichen

Anwendung auf ganze Zahlen und Doubles:

  • short und long sind Synonyme für short int und long int
  • long long ist typischerweise eine ganze Zahl mit 8 Byte
  • unsigned char sind Zahlen von 0, …, 255 (1 Byte)
  • long double (nur diese Form)

Größe eines Datentyps ist maschinenabhängig

Der reservierte Speicherbereich und damit auch der Zahlenbereich für einen einfachen Typ in C ist maschinenabhängig!

  • Zahlenbereiche für konkrete Implementierung in Header-Files definiert
    • limits.h und float.h: Konstanten INT_MAX, INT_MIN, …
  • Alternativ Herausfinden der Größe in Bytes: Operator sizeof
    • Syntax: sizeof(Typ)
  • Es gilt in C:
    • sizeof(unsigned char) \(=\) 1
    • sizeof(short int) \(=\) 2
    • sizeof(short int) \(\le\) sizeof(int) \(\le\) sizeof(long int)
    • sizeof(float) \(\le\) sizeof(double) \(\le\) sizeof(long double)

Konsole: vl02/limits.c

Operatoren (bitweise)

a & b bitweise AND (Löschen von Bits, Maske: 0)
a | b bitweise OR (Setzen von Bits, Maske: 1)
a ^ b bitweise Exklusiv-OR (Invertieren von Bits, Maske: 1)
~a bitweise Negation (Invertieren aller Bits)
a << n alle Bits \(n\) Positionen nach links schieben, Auffüllen mit 0
a >> n alle Bits \(n\) Positionen nach rechts schieben, Auffüllen maschinenabhängig

Operatoren wirken bitweise auf char, short, int, long

Beispiel: Stellen Sie sich einen 8-Bit-Port vor, an dem 8 LED hängen:

  • Wenn Sie gezielt eine LED ausschalten wollen, nutzen Sie ein bitweises AND und in der Maske eine 0 an der Position der LED.
  • Umgedreht für das Einschalten: verODERn Sie den Port-Wert bitweise mit einer Maske, wo an der Position der LED eine 1 steht.

Datenstrukturen in C

Benutzerdefinierte Datentypen in C

  • Strukturen kapseln Daten: Schlüsselwort struct

    struct point {
    int x;
    int y;
} a;

    • Im Beispiel Definition der Struktur point sowie einer Variablen a vom Typ struct point.

    • Bei gleichzeitiger Definition einer Variablen kann auch der Name der Struktur weggelassen werden (“anonyme Struktur”).

    • Bestandteile (“Member”) einer Struktur werden nacheinander im Speicher angeordnet. Unter Umständen gibt es dabei ein “Alignment” (Ausrichtung) der Komponenten an bestimmten Speichergrößen/-blöcken (=> siehe spätere Vorlesung)

  • Variablendefinition

    struct point b;

  • Zugriff auf Daten

    a.x = 10; a.y = 10;
b.x = 42; b.y = 1;

Aufzählungen in C

Aufzählungen in C mit dem Schlüsselwort enum definieren

  • Generelle Struktur:

    enum Typname { Werteliste } Variablenliste;

  • Typname kann weggelassen werden: anonyme Aufzählung
  • Variablenliste kann weggelassen werden
  • Aufzählungselemente werden intern auf int abgebildet
  • (Partielle) Zuweisung von Werten möglich

Konsole: vl02/enum.c

Tagged Unions

/* Aufzaehlung der Basis-Typen */
typedef enum {
    VAL_BOOL, VAL_NUMBER, VAL_OBJ
} ValueType;

/* Tagged Union zur Speicherung der Werte */
typedef struct {
    ValueType type;
    union {
        bool boolean;
        double number;
        Obj* obj;
    } as;
} Value;

double beispiel_get_number(Value value) {
    if (value.type == VAL_NUMBER) return value.as.number;
}
/* Beispiele fuer Zugriff auf eine Tagged Union */

/* Test: Welchen Typ hat ein Value? */
static inline bool is_bool(Value value) {
    return value.type == VAL_BOOL;
}
static inline bool is_number(Value value) {
    return value.type == VAL_NUMBER;
}
static inline bool is_obj(Value value) {
    return value.type == VAL_OBJ;
}

/* Getter: Entpacke den Inhalt eines Values */
static inline bool as_bool(Value value) {
    return value.as.boolean;
}
static inline double as_number(Value value) {
    return value.as.number;
}
static inline Obj* as_obj(Value value) {
    return value.as.obj;
}

/* "Setter": Verpacke einen Wert als neues Value */
static inline Value bool_val(bool value) {
    return (Value){ VAL_BOOL, { .boolean = value } };
}
static inline Value bool_number(double value) {
    return (Value){ VAL_NUMBER, { .number = value } };
}
static inline Value bool_obj(Obj* value) {
    return (Value){ VAL_OBJ, { .obj = value } };
}

Quelle: nach craftinginterpreters.com/types-of-values.html

Erinnerung: In einer Union kann man verschiedene Werte/Typen speichern, aber zu einer Zeit immer nur einen Wert. Deshalb ist eine Union im Speicher immer so breit wie der längste enthaltene Typ. Im obigen Beispiel enthält die innere Union as drei mögliche Typen: bool, double oder einen Pointer auf ein Obj. Die Breite im Speicher ist so bemessen, dass Werte aller drei Typen in der Union Platz finden.

Value x; x.as.number = 42.0; speichert einen Double-Wert in x.as. Das hindert aber nicht daran, später stattdessen das Feld x.as.bool abzufragen. Dabei wird der Wert 42.0 in der Union einfach nur als Boolean interpretiert. D.h. man muss sich merken, was man in der Union abgelegt hat und den richtigen Slot abfragen.

Dazu führt man entweder außerhalb einer Union Buch darüber, welchen Typ eine Union-Variable gerade speichert, oder man arbeitet wie im obigen Beispiel mit einem Aufzählungstyp. Dabei werden der Aufzählungstyp und die Union zusammen in einer Struct gespeichert. Man kann also jederzeit den inneren Typ einer Value-Variablen x abfragen: x.type und dann entsprechend das korrekte Feld der inneren Union x.as zugreifen.

Der Programmierer ist im obigen Beispiel selbst dafür zuständig, dass der gespeicherte Typ Value.type zum Wert in der inneren Union Value.as passt. Deshalb erledigt man den Zugriff typischerweise über eine Reihe von Hilfsfunktionen (oben zusätzlich als static (Begrenzung der Sichtbarkeit auf die Datei, in der die Definition steht – vgl. VL03) und zugleich als inline (der Compiler darf Aufrufe der Funktionen durch den Funktionskörper selbst ersetzen und spart damit den Laufzeit-Overhead bei Funktionsaufrufen – siehe unten) deklariert).

Anmerkung: Obj* value bedeutet, dass value ein Pointer auf ein “Objekt” vom Typ Obj ist, d.h. die Adresse eines Obj speichert. Das müssen Sie hier lediglich zur Kenntnis nehmen – wir sprechen in VL03 und VL04 noch ausführlich über dieses Konzept.

Einfache Arrays in C

  • Allgemeine Syntax: Typ name[ANZAHL];
  • Initialisierung:
    • direkt bei Deklaration: Typ name[ANZAHL] = {wert1, wert2, ...};
    • später: name[i] = wert;

Hinweise

  • Bei Initialisierung bei Deklaration kann (und sollte!) die Anzahl weggelassen werden: Die Größe des Arrays bestimmt sich dann durch die Länge der übergebenen Initialisierungliste. Der Compiler kann viel besser zählen :-)

    • Falls Anzahl angegeben und weniger Werte als ANZAHL: Restliche Werte werden automatisch mit 0 initialisiert …

  • ANSI-C: Anzahl der Arrayelemente muss zur Übersetzungszeit bekannt sein! Ab C11 kann die Arraygröße auch zur Laufzeit berechnet werden (variable Arraylänge: “variable-length array” bzw. VLA)1.

    Anmerkung: Laut wikipedia.org: VLA ist es abhängig vom Compiler, wo das Array im Speicher angelegt wird. Beim GCC scheint es auf dem Stack angelegt zu werden.

  • Überschreiten des Indexbereiches beim Arrayzugriff muss durch den Programmierer abgefangen werden!

  • Größe eines Array x berechnen: sizeof(x)/sizeof(x[0])

    • sizeof(x) würde die Gesamtgröße des Arrays berechnen!

  • char-Array füllen:

    char c[100];
strcpy(c, "hello world :)");  /* man 3 strcpy */
strcpy(c, "wuppie");

Funktionen in C

Funktionen in C

  • Funktionen sind mit Methoden in Java vergleichbar

    => sind aber unabhängig von Klassen bzw. Objekten

  • Syntax:

    Rueckgabetyp Funktionsname(Parameterliste) {
    Anweisungen (Implementierung)
}

  • Aufruf: Nennung des Namens (mit Argumenten) im Programmcode

    int x = foo(42);
  • Unterschied “Parameter” und “Argument”:
    • Funktion hat “Parameter” in ihrer Parameterliste, auch “formale Parameter” genannt
    • Beim Aufruf werden “Argumente” übergeben, auch “aktuelle Parameter” genannt
    In der Praxis verwendet man beide Begriffe i.d.R. synonym.
  • Anmerkungen zum Beispiel:
    • Compiler “liest” Quellcode von oben nach unten
    • Funktionen müssen vor ihrer Verwendung deklariert sein, d.h. es muss zumindest ihre Signatur bekannt sein

Konsole: vl02/simplefunction.c, vl02/simplefunction2.c

Funktionen: Deklaration vs. Definition

  • Deklaration: (Funktions-) Prototyp: Festlegen von Signatur (d.h. Funktionsname und Anzahl, Typ, Reihenfolge der Parameter) u. Rückgabetyp

    void machWas(int, int);

  • Definition: Implementierung der Funktion

    void machWas(int a, int b) {
    printf("a: %d\n", a);
    printf("b: %d\n", b);
}
  • Compiler “liest” Quellcode von oben nach unten
  • Funktionen müssen vor ihrer Verwendung deklariert sein

Allgemein:

  • Deklaration: Macht einen Namen bekannt und legt den Typ der Variablen bzw. die Schnittstelle der Funktionen fest.
  • Definition: Deklaration plus Reservierung von Speicherplatz für die Variable oder Implementierung einer Funktion/Struktur/…

Konsole: vl02/simplefunction3.c, vl02/simplefunction4.c

One Definition Rule (für Funktionen)

Jede Funktion darf im gesamten Programm nur einmal definiert sein!

Überladen von Funktionen gibt es in C nicht!

Funktionen und Parameter

  • Funktionen “ohne” Parameter:
    • Leere Parameter-Liste2 oder Schlüsselwort void
    void fkt();
void fkt(void);
  • Funktionen mit Parameter:
    • Deklaration: Variablennamen können weggelassen werden
    • Definition: Variablennamen müssen angegeben werden
    void fkt(int, char);
void fkt(int a, char b);

void fkt(int a, char b) { ... }
  • Anmerkungen zu Rückgabetyp:
    • ANSI-C: Falls kein Rückgabetyp angegeben wird (auch nicht void), nimmt der Compiler implizit int als Rückgabetyp an
    • Dieses Verhalten ist in C99/C11 verboten
      • Manche Compiler halten sich aber nicht daran!

Konsole: vl02/funcvoid.c

Parameterübergabe in C: Call-by-Value

int add_5(int x) {
    x += 5;
    return x;
}

int main() {
    int erg, i=0;
    erg = add_5(i);
}


  • Default in C ist die call-by-value Semantik:
    • Argumente werden bei Übergabe kopiert
    • Ergebniswerte werden bei Rückgabe kopiert
  • Folgen:
    • Keine Seiteneffekte durch Verändern von übergebenen Strukturen
    • Negative Auswirkungen auf Laufzeit bei großen Daten

Ausnahme: Arrays (Verweis auf spätere VL)

Beispiel: vl02/cbv.c

Globale Variablen (externe Variablen)

int a = 1;

int main() {
    extern int a;
}
  • Außerhalb jeder Funktion definierte Variablen
  • Existieren die gesamte Programmlebensdauer über
  • Müssen bei Nutzung in Funktionen als extern deklariert werden

=> Werden auch als externe Variablen bezeichnet

Hinweis: extern für globale Konstanten bedeutet etwas leicht anderes (=> später)

Varianten für Nutzung globaler Variablen in Funktionen:

  • Deklaration der globalen Variable vor Funktionsdefinition:
    • In Funktion als extern deklarieren, oder
    • In Funktion direkt (ohne Deklaration) nutzen …
  • Deklaration der globalen Variable nach Funktionsdefinition:
    • In Funktion als extern deklarieren

Lokale Variablen (automatische Variablen)

int b = 1;

void f() {
    int b = 42;
}

int main() {
    int b = 3;

    {
        int b = 7;
    }
}
  • Innerhalb einer Funktion (oder Blockes) definierte Variablen
    • Gilt auch für Variablen aus Parameterliste

  • Überdecken globale Variablen gleichen Namens

  • Sichtbarkeit:

    • Außerhalb der Funktion/Blockes nicht zugreifbar
    • Beim Betreten der Funktion Reservierung von Speicherplatz für lokale Variablen
    • Dieser wird beim Verlassen des Blockes/Funktion automatisch wieder freigegeben
    • Namen sind nur nach Deklaration und innerhalb des Blockes, in dem sie deklariert wurden, gültig
    • Namen sind auch gültig für innerhalb des Blockes neu angelegte innere Blöcke

    Software Engineering: Vermeiden Sie lokale Namen, die Namen aus einem äußeren Scope überdecken!

=> Werden auch als automatische Variablen bezeichnet

Schlüsselwort static

void foo() {
    static int x = 42;
    x++;
    printf("x=%d\n", x);
}

int main() {
    foo();  foo();  foo();
}
  • Lokale Variablen mit “Gedächtnis”:
    • Definition mit dem vorangestellten Schlüsselwort “static”
    static int callCount;
  • Eigenschaften:
    • Wert bleibt für die folgenden Funktionsaufrufe erhalten
    • Wert kann in der Funktion verändert werden
    • Dennoch: lokale Variable, d.h. von außen nicht sichtbar/gültig

Initialisierung

(Automatische) Initialisierung von Variablen hängt von ihrer Speicherklasse ab!

  • Extern
    • Mit dem Wert 0 oder vorgegebenem Wert
    • Bereits vor Programmstart (im Code enthalten)
  • Automatisch
    • Werden nicht automatisch initialisiert (!)
    • Bei vorgegebenem Wert ab Aufruf der Funktion
  • Statisch
    • Mit dem Wert 0 oder vorgegebenem Wert
    • Ab erstem Aufruf der Funktion

Konsole: vl02/initVar.c

Neuerungen bei Funktionen

  • C99: inline-Funktionen:

    inline void foo() { ... }

    • Aufrufe von inline-Funktionen können vom Compiler wie ein Makro direkt durch den Funktionskörper ersetzt werden

    • Vorteil: Compiler kann Verwendung (Parameter, Rückgabewert) prüfen

    Wird gern in der Programmierung von Embedded Systems eingesetzt, weil dadurch der eigentliche Funktionsaufruf und der damit verbundene Overhead (Ablegen der aktuellen Registerwerte auf dem Stack, Übergabe der Argumente auf dem Stack, Sprung in den Funktionscode, Abrufen der Parameter vom Stack, …) vermieden wird. (=> Vorgriff Speicherverwaltung)

    Bei Makros findet nur die Textersetzung statt, d.h. es findet dabei keine Prüfung statt, ob das Makro korrekt genutzt wird. (=> Vorgriff Präprozessor und Makros)

  • C11: Funktionen ohne Wiederkehr: _Noreturn

    _Noreturn void exit(int status);

    Sinnvoll, falls Funktion nie zum Aufrufer zurückkehrt, weil sie beispielsweise das Programm beendet. Durch die Markierung kann der Compiler zusätzliche Prüfungen, beispielsweise auf nicht erreichbaren Code hinter der Funktion, vornehmen.

Zusammenfassung

tl;dr

  • Wichtigste Unterschiede zu Java

    • Kontrollfluss wie in Java
    • Basisdatentypen vorhanden
    • Typmodifikatoren zur Steuerung des Speicherbedarfs/Wertebereich
    • Kein bool (erst ab C99 als Makro) vorhanden
    • Integer können im booleschen Kontext ausgewertet werden
    • printf() stammt aus stdio.h, erinnert an java.util.Formatter.format()

  • Strukturen: Kapselung von Daten mit struct

    • Schlüsselwort struct muss “mitgeschleppt” werden
    • Alternative: Definition eines neuen Typen mit typedef

  • Aufzählungen, einfache Arrays

    • Arrays haben eine feste Länge
      • bei Deklaration als konstanten Ausdruck angeben (ANSI-C)
      • oder direkt bei Deklaration Initialisierung mit Initialisierungsliste und Compiler zählen lassen
      • ab C11: Größe des Arrays zur Laufzeit berechnen

  • Funktionen: Kapselung von Operationen

    • Funktionen in C analog zu Methoden in Java
    • Existieren unabhängig von Klassen (gibt in C auch keine)

  • Deklaration vs. Definition, Funktionsprototypen, One-Definition-Rule

  • Parameterübergabe: call-by-value (Standard in C)

  • Unterscheidung globale, lokale und lokale statische Variablen
